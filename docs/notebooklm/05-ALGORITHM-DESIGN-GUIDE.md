# Приложениеге алгоритм құру — практикалық нұсқаулық

> NotebookLM дереккөзі. Өнім-фичалардағы алгоритмдер: матчинг, ұсыныс, іздеу,
> кесте/шектеу. RASPIS (кесте-қозғалтқыш) мен Toi Like (Smart Box) тәжірибесі.

## 1. Алгоритм ойлауының негізгі тәртібі
1. **Есепті нақтыла:** кіріс не, шығыс не, шектеулер қандай, «жақсы шешім» деген не.
2. **Мінсіз шешім керек пе, әлде «жеткілікті жақсы» ма?** Көп өнім-есеп NP-hard
   (кесте, матчинг) — мінсізді іздеу секундтарды жейді. Greedy + repair әдетте
   жеткілікті әрі 100 есе жылдам.
3. **Өлшенетін мақсат-функция жаз.** «Жақсы» деген субъективті болмасын —
   санмен өлшенсін (RASPIS: тесік, pedViol, баланс, сапа).
4. **Шеткі жағдайларды алдын ала ойла:** бос кіріс, ресурс жетпеу, тең-балл.

## 2. Өнімдегі негізгі алгоритм-кластары

### (а) Матчинг / диспетчинг (Uber, Toi Like Smart Box)
Сұраныс ↔ бос ресурс сәйкестендіру. Үлгі: **сүзгі (hard constraints) → скоринг
(0-100 балл, салмақты факторлар) → таңдау → конфликт-тексеру**. TTL-hold (уақытша
ұстау) + rollback — қос-таңдауды болдырмайды.

### (ә) Шектеу-қанағаттандыру (кесте, орналастыру — RASPIS)
Көп шектеуді бір мезгілде қанағаттандыру. Үлгі: **приоритет-кезек (қиынды
бірінші) → greedy орналастыру → repair (сыймағанды жылжыт) → көп-нұсқа іріктеу**.
Кепілдік қабаттары: негізгі 95%-ды шешеді, «guarantee» пасстар қалғанын жабады.

### (б) Ранжирлеу / іздеу
Релеванттылық = салмақты факторлар қосындысы (мәтін-сәйкестік, рейтинг,
жақындық, жаңалық). Bayesian орта (аз дауыспен жоғары рейтинг алдамасын).
Пагинация — keyset (offset емес).

### (в) Ұсыныс жүйесі (recommendation)
Бастау: қарапайым «бірге сатып алынған» (co-occurrence) матрицасы — LLM емес,
арзан әрі тұрақты. Кейін: collaborative filtering (ALS), content-based. Суық
старт: жаңа элементке exploration-буст.

## 3. Скоринг формуласын құру (Smart Box/ранжирлеу үлгісі)
```
score = Σ (салмақ_i × нормаланған_фактор_i) − айыппұлдар
```
- Әр факторды 0-1 аралығына нормала (log-шкала үлкен сандарға).
- Салмақтарды КОНФИГ-та ұста (кодта емес) → A/B тестке дайын, өлшеп реттелетін.
- Айыппұлдар (бас тарту, ескіру) — бөлек, ашық.
- **Ақшамен сапаны жоғарылатпа** (Premium тек тең сапада көтерсін) — әйтпесе
  сенім өледі.

## 4. ЕҢ МАҢЫЗДЫ: өлшеу-тәртібі (RASPIS сабағы)
**Гипотеза → ӨЛШЕ → түзет → ҚАЙТА ӨЛШЕ → регрессияны тексер.**
Алгоритмді «сезіммен» баптама. Нақты деректе (немесе оны имитациялайтын
фаззермен) метрика ал. Осы жобада бірнеше рет «диагноз» қате болып, тек өлшеу
түзетті. Приоритет-иерархияны алдын ала анықта (қайшылықта не жеңеді).

## 5. Фазз-тестілеу (алгоритм-сенімділік)
Ондаған «жасанды кіріс» генераторын жаса (спектрдің екі шеті — ең кіші, ең
үлкен, ең тығыз). RASPIS: 30 жасанды мектеп → крах 0. Toi Like: 30 жасанды
сұраныс. Мақсат: «менің деректерімде жұмыс істейді» жеткіліксіз — «кез келген
деректе құламайды» керек.

## 6. Өнімділік
- Алдымен ДҰРЫС жаса, сосын профильмен оптимизацияла (болжаммен емес).
- Ыстық жол — индекс/кэш. RASPIS: findRoom O(N)→O(ұя), 26с→8с.
- Уақыт-бюджет қой (ауыр есеп шексіз созылмасын) — «жеткілікті жақсы, уақытында».
- Кэштеу + оқиға-негізді инвалидация (дерек өзгергенде тиісті кэш өшеді).

## 7. Жиі қателер
- Мінсіздікті қуып, жылдамдықты жоғалту (пайдаланушы 8с күтпейді).
- Мақсат-функциясыз «жақсарту» (нені жақсартқаныңды өлшемеу).
- Кепілдік қабатсыз: негізгі алгоритм 95% шешеді, қалған 5% — көрінетін бага.
- Rollback дәл болмауы (сәтсіз әрекет күйді бүлдіреді — нақты бүліну болды).
- Салмақтарды кодқа қатыру (A/B, реттеу мүмкін болмайды).
